GUARDIAN API

Integrate security features into your application with the GUARDIAN API. All endpoints return JSON responses.

Beta Notice

The API is currently in beta. Endpoints may change. Currently frontend-only with mock data - full backend coming soon.

Authentication

Include your API key in the request header:

curl -H "Authorization: Bearer YOUR_API_KEY" \
https://api.guardian.security/v1/scan

POST /scan

Perform a comprehensive security scan on a token.

POST /v1/scan

Request Body

{
  "mint": "TokenMintAddress123...",
  "checks": ["honeypot", "rugpull", "whale", "contract"]
}

Response

{
  "success": true,
  "data": {
    "mint": "TokenMintAddress123...",
    "name": "Example Token",
    "symbol": "EXT",
    "risk_score": 23,
    "recommendation": "SAFE",
    "checks": {
      "honeypot": { "passed": true, "buy_tax": 0, "sell_tax": 0 },
      "rugpull": { "passed": true, "liquidity_locked": true },
      "whale": { "passed": true, "top_holder_pct": 12.5 },
      "contract": { "passed": true, "verified": true }
    },
    "risk_factors": [],
    "market_data": {
      "market_cap": 1500000,
      "liquidity": 250000,
      "holders": 1234,
      "age_hours": 168
    }
  }
}

POST /honeypot

Quick honeypot detection - simulates buy/sell to check if token is tradeable.

POST /v1/honeypot

Request Body

{
  "mint": "TokenMintAddress123..."
}

Response

{
  "is_honeypot": false,
  "can_buy": true,
  "can_sell": true,
  "buy_tax": 0,
  "sell_tax": 0,
  "transfer_pausable": false,
  "reason": null
}

GET /threats

Get active threats detected by the GUARDIAN swarm.

GET /v1/threats?status=active&limit=50

{
  "threats": [

      "type": "Honeypot",

GET /v1/whales?token=SOL&limit=20

GET /blacklist

Get known malicious addresses from the Intel database.

GET /v1/blacklist?limit=100

POST /evacuate

Emergency evacuation - transfer all assets from a compromised wallet. Requires wallet signature.

POST /v1/evacuate
Warning

This endpoint transfers ALL assets. Use only in emergencies. Requires signed transaction from source wallet.

Request Body

{
  "source": "CompromisedWalletAddress...",
  "destination": "SafeWalletAddress...",
  "signature": "SignedTransactionBase64..."
}
